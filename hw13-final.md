# 🧠 Реализация интерактивного контроллера управления приложением для личных финансов

## 🎯 Цель

Разработать интерактивный CLI-контроллер для управления бюджетными счетами и транзакциями с использованием библиотеки [`inquirer`](https://www.npmjs.com/package/inquirer). Контроллер должен предоставлять удобный интерфейс для создания, просмотра, редактирования и экспорта данных через консоль.

---

## 📦 Требования к функционалу

### Подготовка

- Все декораторы можно убрать или закомментировать, чтобы решение не затрагивало декораторы
- Все методы строк можно упростить (в классах `Account` и `Transaction`). 
  - Идентификатор можно обрезать
  - В `getSummaryString` можно не выводить информацию о транзакциях
- Добавьте в проект пакет `inquirer`. Этот пакет потребуется для интерактивного взаимодействия в консоли приложения
- Реализуйте класс `ApplicationController` со свойсвом `accountManager` (которое инициализиуется объектом класса `AccountManager`) и методами:
  - `start`
  - `createAccount`
  - `watchAccount`
  - `removeAccount`
  - `removeTransaction`
  - `addTransaction`
  - `exportTransactionsToCSV`
- Предусмотрите возможность начального формирования пары счетов с несколькими транзакциями в каждом счёте

### 1. Главное меню — просмотр всех счетов

- Отображать список всех доступных счетов с краткой информацией (например, название, баланс).
- Позволять выбрать один из счетов для дальнейшей работы.
- Предусмотреть опцию создания нового счёта.
- Предусмотреть опцию выхода из приложения.

### 2. Создание нового счёта

- Запрашивать у пользователя название нового счёта.
- Создавать счёт с уникальным идентификатором и добавлять его в менеджер счетов.
- После создания возвращать пользователя к списку счетов.

### 3. Просмотр выбранного счёта

- Показывать подробную сводку по выбранному счёту.
- Выводить список всех транзакций, связанных с этим счётом.
- Предоставлять меню с действиями:
  - Добавить транзакцию
  - Удалить транзакцию
  - Экспортировать счёт (транзакции) в CSV-файл
  - Удалить счёт
  - Вернуться к списку счетов

### 4. Добавление транзакции к счёту

- Запрашивать у пользователя:
  - Сумму транзакции (числовое значение, больше нуля)
  - Тип транзакции (доход или расход)
  - Дату транзакции (в формате YYYY-MM-DD, по умолчанию текущая дата)
  - Описание транзакции
- Создавать транзакцию с уникальным идентификатором и добавлять её к выбранному счёту.
- После добавления возвращать пользователя к просмотру счёта.

### 5. Удаление транзакции из счёта

- Показывать список транзакций выбранного счёта.
- Позволять выбрать транзакцию для удаления.
- Запрашивать подтверждение удаления.
- После удаления возвращать пользователя к просмотру счёта.
- Если транзакций нет — уведомлять пользователя и возвращать к просмотру счёта.

### 6. Удаление счёта

- Запрашивать подтверждение удаления выбранного счёта.
- При подтверждении удалять счёт и все связанные с ним транзакции.
- После удаления возвращать пользователя к списку счетов.

### 7. Экспорт счёта в CSV

- Запрашивать у пользователя имя файла для экспорта (без расширения).
- Экспортировать все транзакции выбранного счёта в CSV-файл с указанным именем.
- Выводить сообщение об успешном экспорте или ошибке.
- После экспорта возвращать пользователя к просмотру счёта.

## Технические требования

- Использовать пакет [`inquirer`](https://www.npmjs.com/package/inquirer) для интерактивного взаимодействия с пользователем.
- Контроллер реализовать в классе `ApplicationController` с публичным методом `start()`, запускающим интерфейс.
- Все основные операции должны быть реализованы как асинхронные методы класса.
- Использовать классы `AccountManager`, `Account` и `Transaction` для управления данными.
- При возвращении из вложенных меню обеспечивать плавную навигацию назад (в предыдущий пункт меню).
- Обрабатывать некорректный ввод и ошибки.
- При выводе информации использовать понятные и аккуратные форматы (например, `toString()`, `getSummaryString()`).

## Рекомендации по реализации

- Начните с реализации главного меню с выбором счета, создания нового счета и выхода.
- Для каждого счета реализуйте отдельное меню с доступными действиями.
- При добавлении и удалении транзакций используйте списки для выбора и подтверждения.
- Для экспорта в CSV можно использовать встроенные методы класса `Account`.
- Используйте `console.clear()` для обновления экрана и улучшения UX.
